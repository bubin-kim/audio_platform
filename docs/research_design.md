# 차량 경적음 데이터 수집 — 연구 설계 문서

> 교수님 피드백(2026-07-11) 반영: 번호판 중심 기준 각도 수집, 사전 변수 구조화, 거리 범주 근거 확보

---

## 1. 수집 규모 요약

```
차량(2) × 장소(2) × 각도(5) × 거리(3) × 주차위치(2) = 120개 조합
조합당 30회 연속 녹음(자동 커팅) = 총 3,600개 세그먼트
```

- **가용 시간**: 4일 × 5~6시간 = 총 20~24시간
- **일정**: 1일 1(차량×장소) 조합, 30개 세팅/일

---

## 2. 변수 정의 (라벨 스키마)

| 변수명 | 종류 | 값/범위 | 근거·비고 |
|---|---|---|---|
| `angle` | 라벨(필수) | 0 / 45 / 90 / 135 / 180 | 번호판 중심 기준 정면 대비 각도 (교수님 피드백) |
| `distance` | 라벨(필수) | 근거리 / 중거리 / 원거리 | 스마트키 작동거리(제조사 스펙 약 10m) 사전실험으로 최대거리 확인 후 3등분 |
| `vehicle_model` | 라벨(필수) | 차량A / 차량B | 2대 비교 수집 |
| `location_type` | 라벨(필수) | 백화점 / 아파트 | 지하주차장 2종류 — 잔향·배경소음 차이 |
| `park_position` | 라벨(필수) | 전방주차 / 후방주차 | 번호판 노출 방향 |
| `session_date` | 세션 메타 | 수집일(1~4일차) | 재현성 기록 |
| `mic_type` | 세션 메타 | 마이크 모델명 | 장비 특성 기록 |
| `recorder` | 세션 메타 | 녹음자 이름 | 누가 수집했는지 |
| `duration_sec` | 자동 추출 | — | 플랫폼 자동 계산 |
| `sample_rate` | 자동 추출 | — | 플랫폼 자동 계산 |
| `source_start_sec` | 자동 추출 | — | 원본에서 몇 초 지점인지(재현성) |

> **미확정**: `distance`의 실제 미터 값(예: 3m/6m/10m)은 사전실험 결과가 나오면 확정. 그 전까지는 이름만 "근/중/원거리"로 사용.

## 2b. 라벨 코드북 (2026-07-16 확정 — 라벨·파일명 전부 숫자 코드 사용)

플랫폼 라벨과 파일명은 아래 코드로 기록한다. **이 표 없이는 파일명을 해석할 수 없으므로
CSV를 공유할 때 반드시 이 코드북을 함께 전달한다.**

| 필드 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|---|
| `angle` | 0° | 45° | 90° | 135° | 180° | 225° | 270° | 315° |
| `distance` | 근거리 | 중거리 | 원거리 | | | | | |
| `vehicle_model` | 현대 | 기아 | | | | | | |
| `location_type` | 백화점 | 아파트 | | | | | | |
| `park_position` | 전방주차 | 후방주차 | | | | | | |
| `mic_height` | 낮음 | 높음 | | | | | | |

- **파일명 규칙**: `{angle}_{distance}_{vehicle_model}_{location_type}_{park_position}_{mic_height}_{seq:03d}.wav`
  - 예: `2_0_1_1_0_0_001.wav` = 90° · 근거리 · 기아 · 아파트 · 전방주차 · 낮은마이크 · 1번째 조각
  - 끝의 `_001`(seq)은 같은 조합 내 조각 번호 — 파일명 충돌 방지를 위해 필수
- **적용 프로젝트**: 배포 플랫폼 "차량 경적음 테스트" (silence_based, threshold -25dBFS —
  파일럿 horn_pilot_3min.wav 실측 기반, 29조각 검증 완료 2026-07-16)
- 각도가 5개(0~180)에서 **8개(0~315, 45° 간격 전방위)**로, `mic_height`(낮음/높음)가
  라벨로 추가됨 — 조합 수는 8×3×2×2×2×2 = **192조합** (× 조각 30개 = 5,760개 규모;
  실제 수집 범위는 일정에 맞춰 부분 선택 가능)

## 3. 거리 범주 사전실험 계획

- **목적**: 스마트키가 작동하지 않는 최대거리를 찾아 0~그 지점을 3등분
- **참고값**: 제조사 매뉴얼(현대/기아/제네시스) 기준 스마트키 원격 작동거리 약 10m
- **절차안**: 10m 근방을 촘촘하게(5→8→10→12→15m) 테스트하며 "버튼을 눌러도 경적이 안 울리는" 지점 확인
- **판정 기준**: 육안/청각 확인(버튼 조작 결과) — 신호 자체는 RF이므로 음향 SNR과 무관

## 4. 하루 수집 스케줄 (예시: 1일차 — 차량A, 백화점 주차장)

같은 날은 차량·장소가 고정되므로, 남은 변수(각도×거리×주차위치 = 30개)를 아래 순서로 진행하면 차량 재주차를 하루 2회로 최소화할 수 있다.

```
[전방주차]
  거리=근거리: 각도 0→45→90→135→180 (마이크만 이동, 차 고정)
  거리=중거리: 각도 0→45→90→135→180
  거리=원거리: 각도 0→45→90→135→180
[후방주차] (차 재주차)
  거리=근거리: 각도 0→45→90→135→180
  거리=중거리: 각도 0→45→90→135→180
  거리=원거리: 각도 0→45→90→135→180
```

**전체 4일 배정**:

| 일차 | 차량 | 장소 |
|---|---|---|
| 1일차 | 차량A | 백화점 |
| 2일차 | 차량A | 아파트 |
| 3일차 | 차량B | 백화점 |
| 4일차 | 차량B | 아파트 |

## 5. 플랫폼 반영 — Claude Code 프롬프트

```
새 프로젝트를 만들려고 해. 차량 경적음 인식 연구용이야.

label_schema를 이렇게 설정해줘:
- angle: enum [0, 45, 90, 135, 180], required
- distance: enum [근거리, 중거리, 원거리], required
- vehicle_model: string, required
- location_type: enum [백화점, 아파트], required
- park_position: enum [전방주차, 후방주차], required

naming_pattern은 이렇게:
{date}_{vehicle_model}_{location_type}_{park_position}_{distance}_{angle}deg_{seq:03d}

cutting_mode는 fixed_interval로, interval_sec은 경적 울리는 간격에
맞춰 6초 정도로 설정.

이 프로젝트로 조합마다 30초씩 연속 녹음한 걸 업로드하면 자동으로
30개 세그먼트로 잘리고, 이 5개 라벨이 공통으로(common_labels) 붙게
하고 싶어.
```
